Начинать стоит не с “города как в SimCity”, а с микросимулятора, который уже умеет жить сам по себе.

1) Что именно реверсить

Реверс-инжиниринг здесь — это не копирование, а разбор жанра на вопросы:

что видит игрок;
какие скрытые показатели это объясняют;
какие правила связывают действия игрока и реакцию города.
Для SimCity-подобной игры обычно достаточно выделить 5 контуров:

пространство: карта, клетки, зоны;
экономика: деньги, доход, расходы;
популяция: жители, спрос, занятость;
инфраструктура: дороги, доступность, сервисы;
симуляция времени: тик, в котором мир обновляется.
2) Самая примитивная версия

Сделай версию, где есть только:

карта 20×20 клеток;
3 типа клеток: пусто, дорога, дом;
1 ресурс: деньги;
1 сущность: житель;
1 цикл: каждый тик житель либо живет, либо платит налог, либо покидает дом, если нет доступа к дороге/работе/сервису.
Это уже достаточно, чтобы проверить основу жанра: игрок строит, система считает последствия.

3) Минимальная модель данных

На старте не нужны сложные AI-агенты. Нужны простые сущности и правила.

Город

money
population
taxRate
tick
Клетка

type = EMPTY | ROAD | RESIDENTIAL | COMMERCIAL | INDUSTRIAL | SERVICE
level
builtAt
flags / properties
Житель

homeCell
workCell или jobStatus
happiness
needs
alive
Правила

дом создает жильцов;
работа создает доход;
дорога повышает связанность;
сервис повышает счастье;
низкое счастье уменьшает население.
4) Как устроить тик симуляции

Самая важная часть — единый игровой цикл.

Порядок обновления может быть таким:

Игрок применяет действия.
Мир пересчитывает доступность.
Жители обновляют состояние.
Экономика начисляет доход/расход.
Применяются события роста/оттока.
UI получает новый snapshot состояния.
Псевдологика:

for each tick:
applyPlayerActions()
recalculateConnectivity()
updateAgents()
updateEconomy()
resolveConstruction()
generateEvents()
Главное правило: не смешивать UI и симуляцию.
Сначала делай “чистый движок”, потом интерфейс.

5) Архитектура для учебного проекта

Для Java-проекта хорошо подойдет слоистая схема:

Domain

Чистые модели:

City
Tile
Resident
Building
SimulationTick
Simulation Engine

обновление тиков;
расчеты связности;
экономика;
спавн/смерть жителей;
правила роста.
Application Layer

команды игрока: построить, снести, повысить налог;
валидация действий;
сохранение/загрузка.
Presentation

консольная визуализация на первом этапе;
потом GUI или веб-слой.
Такой разрез полезен, потому что потом можно будет заменить графику, не трогая логику.

6) С чего начать по шагам

Этап 1. Песочница без графики

карта в памяти;
построить/снести клетку;
вывести состояние в консоль.
Этап 2. Тики и деньги

каждый тик начислять/списывать деньги;
у зданий есть стоимость строительства и обслуживание.
Этап 3. Простая популяция

дома создают жителей;
жители дают налог;
если денег мало — рост останавливается.
Этап 4. Доступность

BFS/DFS по дорогам;
дом считается “подключенным”, если есть путь к сервису/работе.
Этап 5. Счастье и миграция

у жителя есть happiness;
счастье падает без сервиса/работы/дороги;
при низком счастье житель уезжает.
Этап 6. Баланс и расширение

несколько типов зданий;
зоны;
производство/потребление;
события.
7) Что не делать в первой версии

Не стоит сразу делать:

полноценный pathfinding для всех NPC;
сложную экономику спроса/предложения;
много типов ресурсов;
3D-графику;
“умный” AI;
мультиплеер;
огромную карту.
Это всё убивает MVP. Сначала нужен замкнутый цикл:
построил → мир изменился → увидел эффект → принял следующее решение.

8) Правильная цель первого прототипа

Хорошая первая цель звучит так:

“Игрок может строить дорогу и дом, после чего через несколько тиков появляются жители, начисляются налоги, и при отсутствии связности город деградирует.”

Если это работает, у тебя уже есть ядро city-builder.

9) Практичный способ “реверса”

Не пытайся сразу копировать SimCity. Разбери его как систему:

какие есть метрики;
что влияет на рост города;
какие ограничения заставляют игрока строить инфраструктуру;
какие feedback loops удерживают интерес.
Для каждого механизма задавай вопрос:

что считается входом;
что считается выходом;
как часто пересчитывается;
можно ли выразить это формулой;
можно ли реализовать это в 20–50 строках кода.
10) Самый полезный стартовый каркас

Если нужно совсем коротко, то делай так:

CityEngine.tick()
MapGrid
Building
Resident
Command для действий игрока
SimulationState как единственный источник правды
И начни с одного правила:
дом + дорога + тик + налог.

Это даст тебе рабочую основу, на которую уже можно наращивать всю игру.
