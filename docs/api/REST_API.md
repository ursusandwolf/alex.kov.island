# REST API (планирование)

Управление параметрами симуляции и получение статических данных.

## Базовый URL
`http://localhost:8080/api/v1`

## Эндпоинты

### 1. Управление симуляцией

| Метод | Путь | Описание |
|---|---|---|
| `POST` | `/simulation/start` | Запуск новой симуляции |
| `POST` | `/simulation/stop` | Полная остановка |
| `GET` | `/simulation/config` | Получение текущих настроек |
| `PATCH` | `/simulation/config` | Изменение настроек на лету |

**Пример Start Body:**
```json
{
  "plugin": "nature",
  "width": 100,
  "height": 100,
  "seed": 12345
}
```

### 2. Данные

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/simulation/snapshot` | Получить текущий WorldSnapshot (HTTP-fallback) |
| `GET` | `/simulation/node/{x}/{y}` | Детальная информация о ячейке |
| `GET` | `/plugins` | Список доступных плагинов и их схем настроек |

### 3. Аналитика (история)

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/analytics/history` | Получить историю метрик (для графиков) |
| `GET` | `/analytics/logs` | Последние важные события мира |

## Коды ответов
- `200 OK`: Успех.
- `201 Created`: Симуляция успешно создана.
- `400 Bad Request`: Ошибка в конфигурации.
- `409 Conflict`: Попытка запустить симуляцию, когда она уже активна.
- `503 Service Unavailable`: Движок перегружен или занят.
