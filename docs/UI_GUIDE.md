# Руководство по использованию Web Dashboard

Веб-интерфейс (Dashboard) предназначен для мониторинга симуляции в реальном времени, управления её жизненным циклом и анализа состояния мира.

## 🚀 Запуск
Для полноценной работы необходимо запустить оба компонента:

1. **Backend (Spring Boot)**:
   ```bash
   mvn spring-boot:run -pl island-app
   ```
2. **Frontend (React)**:
   ```bash
   cd island-ui
   npm install
   npm run dev
   ```
Откройте в браузере: `http://localhost:5173`.

## 🕹 Управление симуляцией
В верхней части панели управления доступны следующие кнопки:
- **Start New**: Открывает диалог настройки новой симуляции. Можно выбрать тип (`nature` или `simcity`), размеры сетки и частоту тиков (tick rate).
- **Pause/Resume**: Приостановка и возобновление работы движка симуляции.
- **Stop**: Полная остановка текущей симуляции.

## 📊 Визуализация мира (World Canvas)
Основная часть экрана занята интерактивной картой мира:
- **Отрисовка**: Карта автоматически обновляется через WebSocket (STOMP) после каждого тика симуляции.
- **Выбор ячейки**: Нажмите на любую ячейку карты, чтобы увидеть подробную информацию о ней в боковой панели.
- **Детализация**: В панели деталей отображаются:
    - Список всех видов в ячейке.
    - Количество особей каждого вида.
    - Суммарная биомасса и средний уровень сытости (satiety).

## 💾 Работа с историей и снимками
Система поддерживает сохранение и загрузку состояний:
- **Save Snapshot**: Сохраняет текущее состояние мира в JSON-файл на сервере.
- **History List**: В нижней панели отображается список всех сохраненных снимков.
- **Restore**: Нажмите на снимок в истории, чтобы просмотреть его или запустить новую симуляцию на его основе.

## 🛠 Технические детали
- **API**: Дашборд взаимодействует с бэкендом через REST API (`/api/v1/simulation/**`).
- **Real-time**: Обновления карты приходят по WebSocket на топик `/topic/simulation-updates` (через `SimulationBroadcaster`).
- **Стилизация**: Индикаторы сытости и плотности популяции цветокодированы для быстрого визуального анализа.
