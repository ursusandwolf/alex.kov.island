# Система фаз и приоритетов

Движок симуляции использует детерминированный порядок выполнения задач в рамках одного тика, разделяя их на логические фазы и упорядочивая по приоритетам.

## Порядок выполнения в тике

Один тик проходит через три последовательные фазы:

1.  **PREPARE (Фаза подготовки)**
    *   Обновление глобальных параметров (климат, время суток).
    *   Расчет вспомогательных карт (проводимость дорог, карта загрязнения, привлекательность тайлов).
    *   Подготовка данных, которые понадобятся сущностям в основной фазе.

2.  **SIMULATION (Основная фаза)**
    *   Выполнение логики сущностей (перемещение, питание, размножение).
    *   Работа ECS-систем.
    *   Выполняется параллельно по чанкам (WorkUnits).

3.  **POSTPROCESS (Завершение)**
    *   Сбор статистики за тик.
    *   Очистка погибших сущностей.
    *   Рендеринг (представление) состояния мира.
    *   Создание снимка мира (`WorldSnapshot`).

## Приоритеты (Priority)

Внутри каждой фазы задачи выполняются в порядке убывания приоритета:
- **Высокий приоритет (90-100):** Задачи, результат которых критичен для последующих шагов.
- **Средний приоритет (50-80):** Обычная доменная логика.
- **Низкий приоритет (0-10):** Сбор данных, логирование, визуализация.

### Пример распределения (Nature)
| Задача | Фаза | Приоритет | Описание |
|---|---|---|---|
| `ClimateService` | PREPARE | 100 | Обновляет температуру для всех |
| `AnimalFeedingSystem` | SIMULATION | 80 | Животные едят |
| `AnimalMovementSystem` | SIMULATION | 70 | Животные ходят |
| `CleanupService` | SIMULATION | 10 | Удаляет трупы в конце фазы |
| `ConsoleView` | POSTPROCESS | 0 | Отрисовка в консоль |

## Как задать фазу и приоритет

При реализации интерфейса `ScheduledTask` (или наследовании от `EntitySystem` / `CellService`):

```java
public class MyService implements ScheduledTask {
    @Override
    public Phase phase() {
        return Phase.PREPARE; // По умолчанию SIMULATION
    }

    @Override
    public int priority() {
        return 95; // По умолчанию 50
    }

    @Override
    public void execute(SimulationWorld world, int tickCount) {
        // Логика
    }
}
```

## Рекомендации
1.  **Не делайте всё в SIMULATION:** Если данные можно рассчитать один раз для всех в PREPARE — делайте это там. Это снизит нагрузку на основную фазу.
2.  **Соблюдайте константы:** Используйте именованные константы для приоритетов (см. `TaskRegistry.java`), чтобы видеть общую картину.
3.  **Осторожно с POSTPROCESS:** Задачи в этой фазе не должны менять состояние мира, которое влияет на поведение сущностей в этом же тике.
