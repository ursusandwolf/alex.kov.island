# ADR 004: Интеграция Spring Boot для управления и визуализации

## Статус
Принято (Май 2026)

## Контекст
Проект "Island Ecosystem Simulator" достиг стадии, когда консольный интерфейс становится ограничивающим фактором для визуализации сложных взаимодействий и управления параметрами в реальном времени. Необходимо предоставить API для внешних фронтенд-приложений (React Dashboard) и возможность удаленного управления симуляцией.

## Решение
Трансформировать модуль `island-app` в Spring Boot приложение, которое будет выполнять роль оркестратора и API-шлюза для `island-engine`.

### 1. Архитектурная структура
- **Simulation Management**: `SimulationEngine` и плагины инициализируются как Spring Beans.
- **Control Layer**: REST API для команд управления (START, PAUSE, RESUME, STOP).
- **Observation Layer**: WebSocket (STOMP) для трансляции обновлений `WorldSnapshot` в реальном времени.
- **Persistence Layer**: JSON-сериализация состояния мира через Jackson.

### 2. Технологический стек
- **Spring Boot 3.x**: Web-Starter, WebSocket-Starter.
- **Jackson**: Для маппинга интерфейсов `WorldSnapshot` на конкретные реализации.
- **Spring Profiles**: Для разделения конфигураций (например, `nature` vs `simcity`).

### 3. Дизайн API

#### REST (Control API)
| Метод | Путь | Описание |
| :--- | :--- | :--- |
| `POST` | `/api/v1/simulation/start` | Запуск симуляции с заданными параметрами |
| `POST` | `/api/v1/simulation/pause` | Приостановка тиков |
| `POST` | `/api/v1/simulation/resume` | Возобновление работы |
| `GET` | `/api/v1/simulation/status` | Текущий статус (RUNNING, PAUSED, IDLE) |

#### WebSocket (Observation API)
- **Endpoint**: `/ws-simulation`
- **Topic**: `/topic/world-state` — трансляция `WorldSnapshot` после каждого значимого изменения (тика).

### 4. Управление жизненным циклом
Симуляция запускается в отдельном потоке (управляемом `SimulationEngine`), но контролируется через Spring Service. Это гарантирует, что HTTP-потоки не блокируются логикой симуляции.

## Последствия
### Положительные
- Возможность создания богатого графического UI на React.
- Упрощение интеграции новых инструментов мониторинга.
- Стандартизация управления конфигурациями через `application.yaml`.

### Отрицательные / Риски
- Увеличение времени запуска приложения (overhead Spring Boot).
- Усложнение сериализации из-за использования интерфейсов в `island-engine`.
- Потенциальное давление на GC при частой трансляции больших снимков мира через JSON.
