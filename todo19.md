# Code Review & Plan (2026-04-29)

## Code Review: Island Simulator Refactor

### Общая оценка
Проведен масштабный рефакторинг: система переведена на **Engine-First** архитектуру, внедрен **Data-Driven** подход и оптимизации производительности (LOD, Object Pooling). Код стал значительно чище, Checkstyle соблюден на 100%. Главная задача на завтра — стабилизация баланса, так как текущие параметры все еще приводят к вымиранию экосистемы на длинных дистанциях.

### Находки

#### 🔴 Критично (архитектура / поддержка)
**Нарушение SRP в `Island.java`** — класс все еще выполняет роль и контейнера клеток, и менеджера статистики, и координатора перемещений.
> `public class Island implements SimulationWorld { ... }`
- **Почему**: Класс перегружен методами получения статистики (`getGlobalSatiety`, `getStarvingCount`), которые по логике должны быть в `StatisticsService` или отдельном `ReportingService`.
- **Как исправить**: Перенести методы агрегации данных в `StatisticsService`, оставив в `Island` только управление сеткой и сущностями.

#### 🟡 Стоит улучшить
**LOD лимиты в `FeedingService` и `ReproductionService`** — магические числа.
- **Почему**: Лимиты `30` (repro) и `500` (feeding) захардкожены. При изменении `maxPerCell` в конфиге эти лимиты могут стать неактуальными.
- **Как**: Вынести параметры LOD в `SimulationConstants` или рассчитывать их динамически на основе `maxPerCell` из конфига.

**Метаболизм в `FeedingService`**
- **Почему**: Логика `consumer.consumeEnergy(consumer.getMaxEnergy() * 0.05)` дублирует константы метаболизма.
- **Как**: Использовать `SimulationConstants.BASE_METABOLISM_PERCENT` с множителем штрафа за неудачную охоту.

#### 🟢 Мелочи
- В `EntityContainer.java` метод `fastRemove` использует `indexOf`, что для больших списков превращает удаление в $O(N)$. Поскольку порядок не важен, можно рассмотреть использование `IdentityHashMap` или `Set` для мгновенного поиска индекса/объекта.
- Именование `animal.tryConsumeEnergy` vs `animal.consumeEnergy` — стоит унифицировать контракт.

## План на завтра (2026-04-30)

### 1. Стабилизация экосистемы (Priority: High)
- **Анализ вымирания**: Выяснить, почему на 200-м тике происходит Mass Extinction при 1.5% метаболизме.
- **Механизм "Endangered Protection"**: Доработать Red Book логику в `Island.java`. Если вид близок к вымиранию, дать ему временный иммунитет к голоду или бонус к скорости.
- **Тюнинг растений**: Увеличить массу одного растения, чтобы травоядным не нужно было съедать их сотнями за ход.

### 2. Рефакторинг (Priority: Medium)
- **Reporting Service**: Вынести методы `getGlobalSatiety` и `getStarvingCount` из `Island.java` в `StatisticsService`.
- **LOD Config**: Вынести лимиты семплирования в `SimulationConstants`.

### 3. Новые фичи (Priority: Low)
- **Seasonal Events**: Начать реализацию зимнего периода (снижение метаболизма, исчезновение некоторых видов растений).
- **Genetic Mutation**: Добавить шанс мутации веса или скорости при рождении.

### 4. Тестирование
- Добавить `EcosystemStabilityTest` с проверкой на 1000 тиков.
